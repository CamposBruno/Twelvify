  <action>
    The `landing/` directory currently only contains `code.html`. Scaffold a new Vite + React + TypeScript project there.

    Run from the repo root:
    ```
    npm create vite@latest landing -- --template react-ts
    ```
    This will create `landing/package.json`, `landing/vite.config.ts`, `landing/tsconfig.json`, `landing/tsconfig.node.json`, `landing/src/main.tsx`, `landing/src/App.tsx`, `landing/index.html`.

    Then install Tailwind CSS and its peer dependencies:
    ```
    cd landing && npm install
    npm install -D tailwindcss postcss autoprefixer
    npx tailwindcss init -p
    ```
    This creates `landing/tailwind.config.js` (rename to `.ts` after) and `landing/postcss.config.js`.

    In `landing/src/main.tsx`, ensure `import './index.css'` is present (Vite template includes this).

    Do NOT install additional packages — Tailwind, PostCSS, and Autoprefixer are sufficient for this plan.
  </action>
  <verify>
    Run from `landing/`:
    ```
    npm run build 2>&1 | head -20
    ```
    Should complete without errors. If tailwind config is still `.js`, rename it to `.ts` (content stays the same — just rename the file).
  </verify>
  <done>
    `landing/package.json` exists with `vite`, `react`, `react-dom`, `tailwindcss` listed. `npm run build` completes without errors in the `landing/` directory.
  </done>

<task type="auto">
  <name>Task 2: Configure Tailwind custom theme and zine/punk base styles</name>
  <files>
    landing/tailwind.config.ts

  </files>
  <action>
    Replace `landing/tailwind.config.ts` (or `.js`) with the exact design tokens from `landing/code.html`'s tailwind config block:

    ```typescript
    import type { Config } from 'tailwindcss'

    const config: Config = {
      content: [
        './index.html',
        './src/**/*.{ts,tsx}',
      ],
      darkMode: 'class',
      theme: {
        extend: {
          colors: {
            primary: '#f56060',
            'background-light': '#f8f6f6',
            'background-dark': '#221010',
          },
          fontFamily: {
            display: ["'Permanent Marker'", 'cursive'],
            punk: ["'Special Elite'", 'monospace'],
            body: ["'Inter'", 'sans-serif'],
          },
          borderRadius: {
            DEFAULT: '0px',
            lg: '0px',
            xl: '0px',
            full: '9999px',
          },
        },
      },
      plugins: [],
    }

    export default config
    ```

    Replace `landing/src/index.css` entirely with:

    ```css
    @tailwind base;
    @tailwind components;
    @tailwind utilities;

    @layer base {
      h1, h2, h3 {
        @apply font-display tracking-tight uppercase;
        transform: rotate(-1deg);
      }

      .zine-box {
        @apply border-2 border-slate-900 dark:border-white shadow-[8px_8px_0px_0px_rgba(0,0,0,1)] dark:shadow-[8px_8px_0px_0px_rgba(255,255,255,0.2)];
        background-image: radial-gradient(#e5e7eb 0.5px, transparent 0.5px);
        background-size: 10px 10px;
      }

      .paper-tear {
        clip-path: polygon(0% 0%, 100% 0%, 100% 95%, 98% 98%, 95% 94%, 92% 99%, 88% 94%, 85% 100%, 80% 95%, 75% 99%, 70% 94%, 65% 100%, 60% 95%, 55% 98%, 50% 94%, 45% 99%, 40% 94%, 35% 100%, 30% 95%, 25% 99%, 20% 94%, 15% 100%, 10% 95%, 5% 99%, 0% 94%);
      }
    }
    ```

    In `landing/src/main.tsx`, ensure the body base classes are applied. Update the root div or modify `main.tsx` to add base classes to the document body, or set them in `index.html` (preferred):
    - In `landing/index.html`, add to `<body>` tag: `class="bg-background-light dark:bg-background-dark font-body text-slate-900 dark:text-slate-100 antialiased overflow-x-hidden selection:bg-primary selection:text-white"`

    Source of truth for all values: `landing/code.html` lines 10-49. Do not deviate.
  </action>

  </verify>
